void CalculateImageCopyRegions(uint32_t width, uint32_t height, uint32_t depth, uint32_t layers, uint32_t mips, VkFormat format, VkBufferImageCopy *copyRegions, size_t *count)¶

Calculates and returns the copyable regions needed in order to copy all of the data for an image of the given dimensions.

Parameters

  • width: The width of the image in pixels.

  • height: The height of the image in pixels.

  • depth: The depth of the image in pixels.

  • layers: The number of layers (slices) in the image; 1 for most types other than cube maps and texture arrays.

  • mips: The number of mip levels.

  • format: The image format.

  • [out] copyRegions: An pre-allocated array of size count that will be filled with the valid VkBufferImageCopy list needed to copy the whole image.

  • [in-out]: count The size of the copyRegions array. Pass in null for copyRegions in order to set this value.

ManagedImage IssueBufferToImageCopyCommands(VkPhysicalDevice physicalDevice, VkDevice device, VkCommandBuffer commandBuffer, VkBuffer srcBuffer, VkBufferImageCopy const *copyRegions, size_t numCopyRegions, VkImageCreateInfo const &imageCreateInfo, VkMemoryPropertyFlags imageProperties, VkImageLayout dstLayout, VkAccessFlags dstAccessFlags, VkPipelineStageFlags dstStage)¶

Copies a portion (or portions) of a given VkBuffer to a newly constructed ManagedImage.

Return

A newly constructed ManagedImage with the contents of the srcBuffer as specified by the copyRegions.

Parameters

  • physicalDevice: The VkPhysicalDevice for the given Vulkan objects.

  • device: The VkDevice for the given Vulkan objects.

  • commandBuffer: A VkCommandBuffer that will be used to execute the copy commands.

  • srcBuffer: The VkBuffer to copy data from.

  • copyRegions: The image regions that should be copied to the VkImage.

  • numCopyRegions: The number of copyRegions.

  • imageCreateInfo: Create info for the newly created VkImage.

  • imageProperties: VkMemoryPropertyFlags for the newly created VkImage.

  • VkImageLayout: The image layout the newly created VkImage should be transitioned to.
