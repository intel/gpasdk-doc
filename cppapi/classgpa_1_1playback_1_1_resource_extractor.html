class ResourceExtractor : public gpa::playback::CallableReceiver, public gpa::playback::CallableBroadcaster¶

Provides access to resource data for a captured range

To extract raw binary resource data for a given

ResourceInfo, determine

which ResourceInfo(s) to extract using the

MetadataExtractor. In this

example we’re looking for

RenderTargetInfo objects bound at call index

128. To find

RenderTargetInfo objects we’ll use GRAPHICS_OUTPUT for our ResourceInfo::Usage. As we encounter RenderTargetInfo objects we’ll fill

out the necessary fields in a std::vector<> of

Request::Info objects.

Note that a single

Request may have Request::Infos for any number of

ResourceInfos, call indices, and ResourceInfo::Usages. Also note that a

ResourceInfo::Usage of UNKNOWN will result in the Request::Info being

expanded out to one

Request::Info for each ResourceInfo::Usage. 

auto usage = playback::Resource::Info::Usage::GRAPHICS_OUTPUT;
auto callIndex = 128;
std::vector<ResourceExtractor::Request::Info> requestInfos;
for (auto const& resourceInfo : playbackEngine.GetMetadataExtractor().GetResourceInfos(usage, callIndex)) {
    for (auto const& resourceInfoEx : resourceInfo.GetResourceInfoExs(usage, callIndex)) {
        if (resourceInfoEx.GetTypeId() == ResourceInfo::Ex::GetTypeId<RenderTargetInfo>()) {
            ResourceExtractor::Request::Info requestInfo { };
            requestInfo.usage = usage;
            requestInfo.callIndex = callIndex;
            requestInfo.pResourceInfo = &resourceInfo;
            requestInfos.push_back(requestInfo);
        }
    }
}

Once we have all of the

Request::Info objects that we’re interested in we

need to fill out the necessary fields in

ResourceExtractor::Request. We

start by filling out the number of

Request::Info objects and a pointer to

our collection of Request::Infos. Next we need the storage requirements

for the

Request. This can be obtained using GetRequestStorageSize().

Then we allocate a buffer large enough to hold the

Result data that will

extracted during execution. We need to fill out the

Request object’s

resultStorageSize and pResultStorage fields. Finally we call Execute()

on our

PlaybackEngine with the Request and prepare to process the Result.

Note that the storage allocated here must persist as long as the

Result

obtained from Execute() is in use. The

Result object’s pData field will

simply be the pointer provided for the

Request object’s pResultStorage. 
ResourceExtractor::Request request { };
request.infoCount = requestInfos.size();
request.pInfos = requestInfos.data();
request.resultStorageSize = playbackEngine.GetResourceExtractor().GetRequestStorageSize(request);
std::vector<uint8_t> resultStorage(request.storageSize);
request.pResultStorage = resultStorage.data();
ResourceExtractor::Result result = playbackEngine.Execute(request);

Once we have the

Result we process it by looping over its Result::Info

array. Note that a single

Request::Info may generate 0-n Result::Infos.

Each

Result::Info will have a pointer to the ResourceInfo that data was

extracted for; this is the same

ResourceInfo present in the associated Request::Info (which is also provided as a field in the Result::Info).

Additionaly any

ResourceInfo::Ex is provided as well. 
for (size_t i = 0; i < result.infoCount; ++i) {
    ResourceExtractor::Result::Info const& resultInfo = result.pInfos[i];
    if (resultInfo.state == ResourceExtractor::Result::State::SUCCESS) {
        ProcessResultInfo(resultInfo.pResourceInfo, resultInfo.dataSize, resultInfo.pData);
    }
}
