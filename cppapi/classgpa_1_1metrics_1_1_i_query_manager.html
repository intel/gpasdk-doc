Class Documentation¶

class IQueryManager¶

Interface used to accurately query all metrics for a graphics API call or range of calls.

Note

The underlying implementation used to collect the metrics is based on the QueryManagerType specified when CreateQueryManager was called, and platform specific selection logic also if it was kUnspecified or kDefault.

Note

High-level usage and typical function call flow: 

//!         <APPLICATION STARTUP>
//!             LinkAndPrepareForMdApiSourceUsage (if Intel MD library based query provider will be used)
//!         <WORKLOAD STARTUP>
//!             GetQueryManagerRequiredGraphicsApiExtensions
//!             CreateQueryManager
//!             RegisterCallback
//!             AreRangeQueriesSupported
//!             AreRangeQueriesAcrossCommandBuffersSupported
//!         <WORKLOAD FRAME (iterate as needed)>
//!             <WORKLOAD COLLECTION RANGE with QUEUE CHANGE (iterate based on ranges split by queue changes)>
//!                 OnCreateCommandQueue
//!                 OnCreateCommandList
//!                 Subscribe
//!                 <WORKLOAD CALL / COLLECTION RANGE without QUEUE CHANGE (iterate based on queries subscribed/needed)>
//!                     BeginQuery
//!                     <WORKLOAD GRAPHICS API CALL / CALLS>
//!                     EndQuery
//!                 OnExecuteCommandLists
//!                 <WORKLOAD FRAME RENDER / SUBMIT>
//!                 Unsubscribe
//!         <WORKLOAD SHUTDOWN>
//!             DestroyQueryManager
//!         <APPLICATION SHUTDOWN>
//!             UnlinkMdApiSourceUsage (if Intel MD library based query provider was used)
//!

Note

This interface is NOT thread-safe.

Public Functions

virtual char const *GetInformation() = 0¶

Obtain information about the metrics implementation that would be used.

Return

Pointer to null-terminated buffer containing the name and version of the implementation.

virtual Result Initialize(QueryManagerData *pData) = 0¶

Prepares the QueryManager for usage.

Must be called first, and is automatically called by CreateQueryManager.

Return

Upon success, returns Result::kSuccess. Otherwise, another enumerator is returned and if the provided QueryManagerData::errMessage field is valid, it will contain the cause of the failure.

Note

If the QueryManager implementation is Intel MD library based, the LinkAndPrepareForMdApiSourceUsage function must have been previously called.

Parameters

  • [inout] pData: Pointer to a structure that provides data needed to perform initialization.

virtual QueryManagerType Type() const = 0¶

Returns the type of QueryManager instantiated.

virtual IMetricsSource const *MetricsSource() const = 0¶

Return an interface describing the source of data for this QueryManager.

Note

This will replace the various methods in this class with overlapping functionality.

virtual uint32_t NumAvailableMetrics() = 0¶

This returned number can used to determine the maximum uint32_t supported: validuint32_tRange = [0, NumAvailableMetrics()-1)

virtual MetricDesc const &MetricDescription(uint32_t metricIndex) = 0¶

Gets the metric description for the specified index.

Parameters

  • [in] metricIndex: ID specified in the range [0, NumAvailableMetrics()).

virtual uint32_t NumMetricGroups() = 0¶

This returned number can used to determine the maximum MetricGroupId supported: validMetricGroupIdRange = [0, NumMetricGroups()).

virtual IMetricGroup const *Group(uint32_t index) = 0¶
virtual uint32_t NumMetricsInGroup(uint32_t metricGroup) = 0¶

virtual int32_t GroupContainingMetric(uint32_t metricId) = 0¶

Returns the group id containing the specified metric.

virtual uint32_t DefaultMetricId() const = 0¶

Query the manager for its “default” metric index.

Note

This is essentially what the metric index is for GPU time elapsed.

virtual Result RegisterCallback(QueryResultAvailableCallback callback, void *userData) = 0¶

Used to register a callback function that will be called when metrics are available from the query implementation and Unsubscribe is called.

The results returned to the callback are always returned in the order queried from the CPU. This means that if two back-to-back queries happen on two different command buffers, the results will be returned in the order that BeginQuery and EndQuery were called, not the order the command buffers were executed in.

Parameters

  • [in] callback: Callback function pointer that will receive the results.

  • [in] userData: Opaque user data value that provided to the callback function when it is invoked.

virtual Result Subscribe(CommandBufferData const &bufferData, uint32_t metricGroup, uint32_t numQueries) = 0¶

Subscribe to a group/list of metrics.

Allocates numQueries number of queries indexed in range [0,numQueries). The indices are to be used in BeginQuery and EndQuery.

Parameters

  • [in] bufferData: Provides the graphics command queue to collect metrics from. The OnCreateCommandQueue function must have been previously called for the queue.

  • [in] metricGroup: Metric group index to use in range [0, NumMetricGroups()).

  • [in] numQueries: Maximum number of queries desired.

virtual Result Unsubscribe(CommandBufferData const &bufferData) = 0¶

Unsubscribe from the current group/list of previously subscribed metrics and get all of the resulting metrics collected.

Note

The metrics are returned using the callback function previously provided in an earlier call to RegisterCallback.

Parameters

  • [in] bufferData: Provides the graphics command queue metrics were collected from. Must match the one specified in the previous call to Subscribe.

virtual Result BeginQuery(CommandBufferData const &bufferData, uint32_t queryId) = 0¶

Begin a query.

Identifies the start of a query using the specified command buffer data for a specific query index.

Note

There must be an unspecified/available query of those requested in an earlier call to Subscribe.

Note

Currently, only one query is allowed to be open/active at a time thus nested queries within the same command buffer are not supported. After calling BeginQuery, EndQuery must be called before BeginQuery is called again.

Parameters

  • [in] bufferData: Graphics command queue and command buffer state to begin the query with. The Subscribe function must have been previously called for the queue. For Vulkan, the VkCommandBuffer must be in the recording state.

  • [in] queryId: Query index to associate the query with [0, numQueries) where numQueries was specified earlier in Subscribe. Query indices are not required to be used sequentially.

virtual Result EndQuery(CommandBufferData const &bufferData, uint32_t queryId) = 0¶

End a query.

Identifies the end of a query using the specified command buffer data for the specified query index.

Parameters

  • [in] bufferData: Graphics command queue and command buffer state to end the query with. The queue must match the one provided in the prior Subscribe and BeginQuery calls. For Vulkan, the VkCommandBuffer must be in the recording state.

  • [in] queryId: Index of query to end that must match the one provided in the prior BeginQuery call.

virtual Result OnCreateCommandQueue(void const *commandQueue) = 0¶

Creates graphics command queue based resources needed by the QueryManager.

Must be called after creating any graphics command queue and before calling Subscribe.

Note

Resources created by calling this function are automatically released when the QueryManager implementation is destroyed.

Parameters

  • [in] commandQueue: Graphics command queue that will be used to collect metrics from.

virtual Result OnCreateCommandList(CommandBufferData const &commandList) = 0¶

Creates graphics command buffer based resources needed by the QueryManager.

Must be called after creating any graphics command buffer (and encoder if the graphics API is Metal) and before calling OnExecuteCommandLists.

Note

Resources created by calling this function are automatically released when the QueryManager implementation is destroyed.

Parameters

  • [in] commandList: Graphics command queue and command buffer (as well as the command encoder if the graphics API is Metal). The OnCreateCommandQueue function must have been previously called for the queue.

virtual Result OnExecuteCommandLists(void const *commandQueue) = 0¶

Perform implementation specific logic, such as setting the metric group/list configuration metrics will be collected for.

Must be called right before executing/submitting/committing the queue or command buffer(s) to the GPU to be rendered.

Parameters

  • [in] commandQueue: Graphics command queue associated with the work about to be submitted to the GPU. An earlier call to the OnCreateCommandList function must have included the specified queue, as well as a prior call to the Subscribe function.

virtual bool AreRangeQueriesSupported() = 0¶

Returns whether or not the query provider supports querying a region (consecutive graphics API calls).

virtual bool AreRangeQueriesAcrossCommandBuffersSupported() = 0¶

Return whether or not the query provider supports querying a region (consecutive graphics API calls) that spans more than one graphics command buffer (begin query is in one grapics command buffer and the end query is in a different one).
